标注格式说明（Pascal VOC XML）

每张图片对应一个 XML 文件，支持同一张图中标注多个目标框。

单目标示例（000003.xml）
<annotation> <filename>000003.jpg</filename> <size> <width>891</width> <height>501</height> <depth>3</depth> </size> <object> <name>garbage_bin</name> <bndbox> <xmin>180</xmin> <ymin>120</ymin> <xmax>350</xmax> <ymax>460</ymax> </bndbox> </object> </annotation>
多目标示例（000010.xml）
<annotation> <filename>000010.jpg</filename> <size> <width>499</width> <height>346</height> </size> <object> <name>overflow</name> <bndbox>...</bndbox> </object> <object> <name>garbage_bin</name> <bndbox>...</bndbox> </object> <object> <name>garbage</name> <bndbox>...</bndbox> </object> </annotation>

VOC 格式转换为 YOLO 格式

YOLOv8 需要 YOLO 格式标注（归一化中心坐标），运行转换脚本自动完成。

VOC 格式（绝对像素坐标）
xmin=90, ymin=166, xmax=166, ymax=260 图像大小: 499 × 346 类别: overflow (id=1)
YOLO 格式（归一化中心坐标）
1 0.256513 0.614740 0.152304 0.272254 # 格式: class_id cx cy w h # 坐标均除以图像宽高，归一化到 [0,1]
一键转换命令
# 第一步：格式转换（VOC XML → YOLO txt + dataset.yaml） py -3 D:\garbage_system\convert_voc2yolo.py # 第二步：开始训练（需先安装：pip install ultralytics） py -3 D:\garbage_system\train_yolo.py # 第三步：仅验证已训练模型 py -3 D:\garbage_system\train_yolo.py --mode=val

训练流程

1
数据准备
运行 convert_voc2yolo.py，将 VOC XML 标注转换为 YOLO txt 格式， 按 8:2 自动分割 train/val，生成 dataset.yaml 配置文件。
2
模型训练
运行 train_yolo.py，基于 YOLOv8n 预训练权重进行迁移学习， 100 轮训练加 20 轮早停，自动数据增强（mosaic、flip、旋转）。
3
部署推理
训练完成后 best.pt 自动复制到系统路径，重启 app.py 即可加载真实模型，替换演示模式。

类别定义与预警规则

YOLO ID VOC 标签 中文名称 是否触发预警 说明
0 garbage_bin 垃圾桶 正常 检测垃圾桶位置，用于判断溢出关联
1 overflow 垃圾溢出 触发预警 垃圾堆积超出桶口，需立即处理
2 garbage 散落垃圾 触发预警 桶周围地面有散落垃圾，影响环境卫生
3 fire 火焰 触发预警 检测到明火，最高优先级预警
4 smoke 烟雾 触发预警 检测到烟雾，次高优先级预警
数据集扩展建议： 可继续添加 hazardous（有害垃圾） 等类别， 使用 LabelImg 工具标注后重新训练即可扩展检测能力。
